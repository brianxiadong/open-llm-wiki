# 内部知识库「使用小助手」内容包

面向**最终用户**的使用说明 Wiki 页面，供导入到公开知识库，方便同事自助答疑。

## 管理员如何上架

1. 使用管理员或指定账号登录，**新建一个知识库**（建议名称含「使用帮助」「小助手」等，便于识别）。
2. 在知识库 **设置** 中勾选 **公开**，保存。
3. 在同一页面的 **导入 Wiki（ZIP）** 中，上传由本目录打包的 zip（见下），模式选 **合并** 即可（若希望完全以本手册为准，可选替换模式，会清空原有 Wiki 页面，请谨慎）。
4. 等待 **索引重建** 任务完成后再对外推荐使用。

## 打包命令（在仓库根目录执行）

```bash
cd seed/internal-kb-assistant && zip -r ../../internal-kb-assistant-wiki.zip wiki
```

将生成的 `internal-kb-assistant-wiki.zip` 在设置页上传即可。

## 内容说明

- 全部页面在 `wiki/` 下，语言面向**业务用户**，不涉及部署、接口、底层组件等技术细节。
- 可根据本单位制度在导入后由管理员在网页上**直接改 Wiki 页面**做本地化补充。
